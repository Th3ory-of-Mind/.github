# Support

Public repositories are community projects. Documentation, issue forms, and
existing discussions are the primary support resources.

Use the bug form for reproducible defects and GitHub Discussions when enabled
for usage questions. Include the repository, version or commit, environment,
expected result, actual result, and a minimal synthetic reproduction.

Do not post secrets, credentials, personal information, production data, or
unredacted logs. Suspected vulnerabilities must follow [SECURITY.md](SECURITY.md)
instead of a public issue.

Support is provided on a best-effort basis. Opening an issue does not establish
a service-level agreement, maintenance commitment, compatibility guarantee, or
release schedule. Repository documentation and release notes define the
currently published support surface.
