# Contributing

Thank you for helping improve Th3ory of Mind. Contributions should be small,
reviewable, and consistent with the repository's documented scope.

## Before opening a change

Search existing issues and choose the matching issue form. Discuss substantial
architecture or contract changes before implementation. Never put secrets,
credentials, personal information, or production data in an issue, commit,
test fixture, log, or pull request.

## Pull requests

Create a focused branch, use clear commits, and complete the pull request
template. Add or update tests for behavior that can be validated automatically.
Document provenance for generated or third-party material and preserve license
notices.

A public interface, schema, policy, event, manifest, or compatibility contract
change requires an architecture decision record and a migration plan. Reviewers
may also require rollback instructions and staged adoption evidence.

All changes require human review. Approval and merge authority remain with the
repository maintainers identified through CODEOWNERS and repository settings.

By contributing, you agree that your contribution is licensed under this
repository's license and that you will follow the Code of Conduct.
