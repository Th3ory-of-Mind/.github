# Security Policy

## Report a vulnerability privately

Do not open a public issue for a suspected vulnerability. Use
[GitHub Security Advisories](https://github.com/Th3ory-of-Mind/.github/security/advisories/new)
to submit a private report to the maintainers. If the advisory form is
unavailable, use GitHub's repository reporting interface to contact the
maintainers without publishing technical details.

Include the affected component and version or commit, impact, reproduction
steps, and any suggested mitigation. Use minimal synthetic examples: do not include secrets,
credentials, personal information, or production data.

## What to expect

Maintainers will assess reports according to impact, reproducibility, and the
affected repository's current support state. A maintainer may ask for more
information through the private advisory. Timing, remediation, disclosure, and
credit are determined case by case; this policy does not promise a response or
release deadline.

Public security design questions that do not reveal a vulnerability may use
the security design issue form.
